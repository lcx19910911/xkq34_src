=================物流信息api修改方案=================
针对我们现在查询不到物流信息做了如下修改：
-------------------------------------------------------------------------------------
1 客户自己设置付费的key：程序中首先用客户设置付费的key来订阅单号的物流信息，快递100根据我们订阅的单号推送消息到我们本地，在显示物流信息的过程中，首先到本地的数据库中查询，如果本地没有数据，我们还是会通过调用快递100接口来获取物流信息，如果再获取不到数据的话，我们会提供快递100的html API 链接到快递100网站上查询快递数据
2 客户不设置key，通过我们默认的key来查询物流信息：首先也是通过快递100的API来获取数据，如果获取不到的话任然会提供快递100的html API链接到快递100网站上查询，这种方式的问题是：我们的key一天只能提供2000次的查询，如果超过了次数的话会查询不到快递数据的。
-------------------------------------------------------------------------------------
